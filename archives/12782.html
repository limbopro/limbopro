V. 一份完整有效的 conf 内容示例

完整示例 

hostname = *.pianku.me
#www.pianku.me adsblock
^https:?/\/\b..+pianku.\w{2,3}\b.* url response-body var pycount=10 response-body var pycount=0

完整且合法的 conf 配置包含两个部分：
1.hostname：主机名，.pianku.me，其中星号可代表任意字符、字符串：可以匹配到 m.pianku.me，x.pianku.me 等等；
2.^https:?/\/\b..+pianku.\w{2,3}\b.* 是正则表达式，用来匹配具体请求URL的，这条正则可以匹配到 https://m.pianku.me/py/lJmM2kDN4cza_1.html?158064（视频播放页面）；var pycount=10 匹配 HTTP 响应体中的内容，需要被替换的内容；var pycount=0 为替换内容；^https:?/\/\b..+pianku.\w{2,3}\b.* url response-body var pycount=10 response-body var pycount=0 即将网页元素中 var pycount=10 替换为 var pycount=0；

即你可以将之理解为，先要有主机名（如 pianku.me ），没有的话 Quantumult X 就就没办法进行下一步匹配了；有了主机名之后，就要对具体的URL请求进行相对应的操作，这条URL符合哪些条件呢？就涉及到正则表达式（如 ^https:?/\/\b..+pianku.\w{2,3}\b.* ，用正则表达式的好处在于可以更精细的匹配想要去广告的URL；）；response-body 后的参数 也可以是 正则表达式（是的！虽然 var pycount=0 这一条很普通，但这也是一条表达式！）；

所有匹配^https:?/\/\b..+pianku.\w{2,3}\b.* 的请求（Request），如其 response-body（响应体）中包含var pycount=10的话，则将其替换为var pycount=0；

IX. 去网页广告的2种基本思路

基本思路

禁止加载广告资源

禁止加载广告资源；（使用分流规则的Reject对广告域进行Request禁止；使用重写类型如reject/reject/reject-200.../对匹配的广告域进行Request禁止；）；

重写网页资源

本文文末为大家提供了两种重写网页资源的超便捷方法，通过嵌入JavaScript文件/CSS文件实现对网页资源的控制；

重写网页资源；重写在QuantumultX 对应 Rewrite、 Reponse响应（与之对应的是Request，即请求）内容，在Quantumult X 中响应又分为response-body/script-response-body/script-response-header...等等，每个类型都有其妙用；（本文所讨论的正式 response-body，改写HTTP响应体 ）

我们应该知道的是每个网站网页都有其结构，网页内各个项目的长宽比例都是统一的，网页自顶部开始，标题，播放框，等等；

广告会加载在哪个地方呢？是悬浮的还是固定长宽比例的，是引用的站内资源还是站外资源，这是在去广告过程中应该考量的；

如果广告是动图？那么你应该找到这个动图对应的URL，分析该URL特点，使用分流或重写对其Requeset禁止，这样网页内就不会再出现该Gif了；

    V. 一份完整有效的 conf 内容示例

    完整示例 

    hostname = *.pianku.me
#www.pianku.me adsblock
^https:?/\/\b..+pianku.\w{2,3}\b.* url response-body var pycount=10 response-body var pycount=0

    完整且合法的 conf 配置包含两个部分：
    1.hostname：主机名，.pianku.me，其中星号可代表任意字符、字符串：可以匹配到 m.pianku.me，x.pianku.me 等等；
    2.^https:?/\/\b..+pianku.\w{2,3}\b.* 是正则表达式，用来匹配具体请求URL的，这条正则可以匹配到 https://m.pianku.me/py/lJmM2kDN4cza_1.html?158064（视频播放页面）；var pycount=10 匹配 HTTP 响应体中的内容，需要被替换的内容；var pycount=0 为替换内容；^https:?/\/\b..+pianku.\w{2,3}\b.* url response-body var pycount=10 response-body var pycount=0 即将网页元素中 var pycount=10 替换为 var pycount=0；

    即你可以将之理解为，先要有主机名（如 pianku.me ），没有的话 Quantumult X 就就没办法进行下一步匹配了；有了主机名之后，就要对具体的URL请求进行相对应的操作，这条URL符合哪些条件呢？就涉及到正则表达式（如 ^https:?/\/\b..+pianku.\w{2,3}\b.* ，用正则表达式的好处在于可以更精细的匹配想要去广告的URL；）；response-body 后的参数 也可以是 正则表达式（是的！虽然 var pycount=0 这一条很普通，但这也是一条表达式！）；

    IX. 去网页广告的2种基本思路

    基本思路

    禁止加载广告资源

    禁止加载广告资源；（使用分流规则的Reject对广告域进行Request禁止；使用重写类型如reject/reject/reject-200.../对匹配的广告域进行Request禁止；）；

    重写网页资源

    本文文末为大家提供了两种重写网页资源的超便捷方法，通过嵌入JavaScript文件/CSS文件实现对网页资源的控制；

    重写网页资源；重写在QuantumultX 对应 Rewrite、 Reponse响应（与之对应的是Request，即请求）内容，在Quantumult X 中响应又分为response-body/script-response-body/script-response-header...等等，每个类型都有其妙用；（本文所讨论的正式 response-body，改写HTTP响应体 ）

    我们应该知道的是每个网站网页都有其结构，网页内各个项目的长宽比例都是统一的，网页自顶部开始，标题，播放框，等等；

    广告会加载在哪个地方呢？是悬浮的还是固定长宽比例的，是引用的站内资源还是站外资源，这是在去广告过程中应该考量的；

    如果广告是动图？那么你应该找到这个动图对应的URL，分析该URL特点，使用分流或重写对其Requeset禁止，这样网页内就不会再出现该Gif了；
